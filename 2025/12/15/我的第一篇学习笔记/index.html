✅ 后续常用命令（收藏）

目的 命令
新建文章 hexo new "标题"
本地预览 hexo server
生成静态文件 hexo generate
部署到 GitHub hexo deploy
一键三连 hexo clean && hexo g && hexo d

✅ 换主题（可选）

  1. 去 https://hexo.io/themes 挑一个，比如 hexo-theme-next。
  2. 按主题 README 的指引，通常是：
  1. 修改 _config.yml 里的 theme: next，再 hexo clean && hexo g && hexo d 即可。

✅ 常见问题速查

现象 解决
页面 404 检查仓库名是否 用户名.github.io，是否 Public
样式丢失 确认 _config.yml 里的 url 和 root 没填错
部署后没更新 浏览器强制刷新 Ctrl+F5 或清空缓存

✅ 下一步建议

  1. 把这篇教程收藏到浏览器书签；
  2. 把 my-notes 文件夹整个用 Git 管理（git init + git push 到另一个仓库做备份）；
  3. 每学完一个知识点就 hexo new 一篇，坚持 30 天，博客自然成型。

如果卡在哪一步，直接复制报错信息发给我，我帮你秒回！
